EXISTING AND PROPOSED SYSTEM


 --------------------------------------------------------------------------------
Existing Business Process
Monitoring depends on user complaints and scheduled visits. Environmental problems are detected after service degradation, maintenance is reactive.
                                       
 --------------------------------------------------------------------------------
Proposed Business Process
Monitoring is continuous and centralized. Environmental risks are detected in real time. Alerts are sent automatically. Maintenance becomes preventive rather than reactive.
                                       
--------------------------------------------------------------------------------
   B. GLOBAL DESCRIPTION OF THE PRODUCT


 --------------------------------------------------------------------------------
Main Functionality
The proposed system is an IoT-based monitoring platform designed to supervise environmental conditions inside telecommunication shelters that host GPON equipment. The system addresses the operational gap identified in Form 1, where temperature and vibration are not monitored continuously and issues are detected only after service disruption. The main functionalities are:
 Real-Time Temperature Monitoring
The system continuously measures temperature inside the shelter using dedicated sensors. Data is transmitted to a central server and displayed on a monitoring dashboard. When temperature exceeds predefined limits, the system generates an alert.
 Real-Time Vibration Monitoring
Vibration sensors detect mechanical disturbances that may affect fiber connections and optical modules. Data is analyzed to determine whether vibration levels remain within safe operational limits.
 Threshold-Based Detection
Administrators configure temperature ranges and vibration thresholds. The backend automatically compares incoming data with these parameters to detect abnormal conditions.
 Machine Learning Risk Classification
The system analyzes vibration patterns and environmental trends to classify risk levels. The classification supports early identification of potential impact on GPON equipment reliability.
 CCTV Evidence Capture
When abnormal conditions are detected, the edge device captures a timestamped image from the shelter interior. The image is stored in the server and linked to the alert record.
 Real-Time Notification
Alert notifications are delivered through Telegram to assigned technicians and administrators to support immediate response.
 Centralized Web Dashboard
The dashboard provides current temperature and vibration values, risk level indicators
   Active and historical alerts, CCTV evidence, & Historical trend graphs

The system supports preventive maintenance by enabling early detection before service degradation occurs.

 --------------------------------------------------------------------------------
Constraints 
From the user's perspective, the system has several limitations:
 Internet Connectivity Dependence
Continuous monitoring and notification require stable network access from the shelter to the central server.
 Power Supply Dependence
Monitoring devices require consistent power availability inside the shelter.
 Sensor Accuracy and Calibration
   Temperature and vibration sensors have tolerance limits and require periodic calibration to maintain accuracy.
 Hardware Processing Limits
Edge devices have limited processing capacity compared to enterprise servers.
 Physical Installation Constraints
Limited space inside the shelter restricts device placement and expansion.
 Environmental Exposure
Extreme environmental conditions may affect sensor durability over time.

Detail of the process inside the Activity Diagram and Data flow diagram
sensor data acquisition, reception, and storage :

- The data is stored in the database.

--------------------------------------------------------------------------------
     
Threshold Analysis and Machine Learning
- Temperature is compared with warning and critical thresholds.
- Vibration is compared with the maximum threshold set by the administrator.
- If the value is still normal, the dashboard is only updated with normal status. If the value exceeds the threshold, the system proceeds to risk analysis using Machine Learning.

The machine learning model analyzes:
- Vibration patterns
- Temperature increase trends
- Previous historical data
Classification results:
- Low (Safe)
- Medium (Caution)
- High (Danger)
If the result is High:
- The system generates an alert.
     - Raspberry Pi activates the camera. 
- The camera takes pictures with: Timestamp, Shelter ID, Device ID
- The pictures are sent to the server.
     Notification Delivery Process
- The system sends notifications via Telegram Bot API.
- Technicians receive notifications in real time.

                             D. ALTERNATIVE SOLUTION
                                       
In order to ensure that the selected system is the most appropriate solution, several alternative approaches have been analyzed. Each alternative has  different advantages and disadvantages in dealing with the problem of monitoring telecommunication shelters and protecting GPON equipment reliability.


Alternative Solution 1  -  Smart CCTV System with AI-Based Human Detection

This approach uses CCTV cameras and artificial intelligence to detect human presence inside the shelter. The artificial intelligence model uses various computer vision techniques such as human detection and face recognition in identifying individuals entering the shelter area. The system can differentiate between authorized and unauthorized individuals based on stored identity data. When suspicious activity is detected, the system will automatically alert administrators or technicians along with captured image evidence.
In comparison to the conventional CCTV systems, this solution enhances security awareness by offering automated detection instead of manual video monitoring. The system also enables quicker response to unauthorized access compared to the conventional CCTV systems. 
However, this solution is primarily concerned with visual security monitoring. It does not monitor environmental conditions like temperature and vibration, which are the most significant factors that influence the reliability of GPON equipment reliability. As overheating and high vibration are significant causes of equipment deterioration, this solution alone is not sufficient to support the preventive maintenance goal.

Alternative Solution 2  -  Periodic Data Logging with Offline Analysis

In this method, the data is recorded continuously by the environmental and vibration sensors, and the data is stored locally in a storage device such as an SD card or internal memory. In this method, instead of transmitting the data in real time, the data is collected by the technicians during their visits and analyzed offline.
This  method is relatively simple and inexpensive because it does not require continuous internet connectivity or cloud infrastructure, or real-time server processing. It reduces operational costs and minimizes system complexity.
However, the major limitation of this method is the lack of real-time monitoring and automatic alert systems. In case of sudden increases in temperature or unusual levels of vibration, the fault will not be detected until the next scheduled inspection. Hence, this solution is not appropriate for environments where preventive and immediate response are required.


Alternative Solution 3  -  Threshold-Based IoT Monitoring Without Machine Learning

In this approach, temperature and vibration sensors are placed inside the shelter and continuously transmit data continuously to a central server. The system compares the values sent by the sensors with threshold limits defined by the administrator. When the temperature or vibrations are greater than defined threshold  limits, an alert is sent immediately, and a notification is sent to the assigned technician.
This approach is simple and easier to implement compared to a system that uses a machine learning approach. It does not require a machine learning model or a classification system. It has lower computational requirements and can run on systems with low processing power. The cost of developing this system is also lower compared to a system that uses a machine learning approach.
The disadvantage of this approach is that it will respond only when threshold limits are exceeded and will not analyze trends or patterns over a period of time. It will not be able to classify risk levels or identify warning signs of impending danger. Therefore, this approach will have limited capabilities for preventive maintenance if a machine learning approach is used in the system.

Alternative Solution 4  -  Hybrid Monitoring with Environmental Sensors and Periodic Manual Inspection

In this approach, simple environmental sensors are installed to monitor temperature conditions in the shelter, whereas vibration checks and equipment inspections are performed manually during scheduled maintenance visits. Alerts are issued only for extreme temperature conditions, while vibration checks and equipment conditions are handled manually during scheduled maintenance.
This approach simplifies the complexity of implementing the system, reducing its cost because it does not require advanced vibration analysis or machine learning integration. It also enables manual inspection of the physical condition of the GPON equipment, fibers, and shelter infrastructure during scheduled maintenance.
However, the major limitation of this approach is its response time, which is slow because no real-time analysis of the system's vibrations or inspection of the equipment conditions is performed during scheduled maintenance.As a result, problems arising between scheduled maintenance visits may go undetected, increasing the probability of equipment degradation or service disruption. This approach offers little support for preventive maintenance.
